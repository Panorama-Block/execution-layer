# Strategy Agent Business Value

## The Intelligence Differentiation

Without the Strategy Agent, PanoramaBlock is an action router: "swap X for Y", "stake Z". The user still needs to know what to do.

With the Strategy Agent, PanoramaBlock becomes a financial advisor: "I want yield with moderate risk" -> the system decides what to do.

This is the difference between a tool and a copilot.

## User Retention Through Strategies

Single actions are one-time. Strategies create ongoing relationships:
- User sets up a yield strategy -> monitors it regularly
- Strategy needs rebalancing -> user returns
- Market conditions change -> agent suggests adjustments
- New protocols launch -> agent proposes new strategies

## Revenue Potential

Strategies involve more transactions than single actions:
- Single swap: 1 transaction
- Yield strategy: 4+ transactions (swap + supply + LP + stake)

More transactions = more potential for fee capture (future feature in executor).

## Competitive Moat

Most DeFi interfaces are action-based. Building a strategy intelligence layer creates a moat:
- Strategy data (which strategies work) accumulates over time
- User profiles (risk preferences) become more accurate
- Protocol knowledge (optimal parameters) improves continuously
- Network effects: more users -> better strategy data -> better recommendations
